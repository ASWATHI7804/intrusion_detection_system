Project Overview

This project involves the comprehensive development of a simulated Intrusion Detection System (IDS) using Cisco Packet Tracer. IDS plays a critical role in modern cybersecurity frameworks by monitoring network traffic for potential threats, vulnerabilities, or unauthorized access attempts.

Our focus is on creating a signature-based Network IDS (NIDS) that uses Cisco's IOS features to detect specific patterns in network traffic, such as ping floods or port scans. The simulated system inspects ICMP packets and alerts administrators when a match to a known attack signature is detected. All alerts are forwarded to a SYSLOG server, enabling centralized logging and real-time monitoring of security events.

This project not only illustrates how IDS works in practice but also provides hands-on experience in router configuration, network diagnostics, and threat detection, which are valuable skills in real-world IT infrastructure management.

Network Design

The simulated network topology includes three distinct LANs connected through routers. Each LAN contains multiple end devices such as PCs and laptops, along with a centralized SYSLOG server, and application servers like HTTP and FTP.

  • Router0: Acts as the IDS gateway for Network 1, monitoring inbound traffic.
  • Router1 and Router2: Facilitate inter-network routing and simulate traffic sources.
  • SYSLOG Server: Collects and logs intrusion alerts triggered by the IDS.
  • Client PCs: Generate traffic, including ICMP (ping) packets to test detection.

The IDS is specifically configured on the Router0‚Äôs GigabitEthernet0/0 interface to monitor incoming ICMP traffic using preloaded Cisco IOS signatures. Alerts are forwarded to the SYSLOG server for real-time logging and review.

Configuration Steps

The configuration involves setting up the security features on the router, defining IPS rules, activating signature categories, and configuring logging mechanisms. Below is the detailed process:

Steps:

  1. Enable the security license package: license boot module c1900 technology-package securityk9
  2. Define the IPS configuration file location: ip ips config location flash:
  3. Create a named IPS rule: ip ips name IDS
  4. Enable and activate IOS_BASIC signature category:
    • ip ips signature-category
    • category all ‚Üí retired true
    • category IOS_BASIC ‚Üí retired false
  5. Apply the IPS rule to the incoming interface: interface g0/0 ‚Üí ip ips IDS out
  6. Configure the SYSLOG server:
    • logging host [SYSLOG_IP]
    • service timestamps log datetime msec
    • Enable SYSLOG service in Packet Tracer on the server device

Detection Output

Once the configuration is complete, the IDS becomes active on Router0. When a PC from another network sends ICMP ping requests to a PC in Network 1, the IDS compares these packets with signature ID 2004 (ICMP Echo Request). If matched, the router logs an alert in the CLI and forwards it to the SYSLOG server.

Example Alert Message:

%IPS-4-SIGNATURE: Sig:2004 Subsig:0 Sev:25 [192.168.10.2 -> 192.168.1.2:0] RiskRating:25

These messages confirm that:

  • The IDS signature engine is active and scanning traffic.
  • Alerts are successfully triggered for suspicious activity.
  • Logs are being forwarded to the SYSLOG server for auditing and review.

Working of the System

The IDS works by integrating Cisco IOS‚Äôs Intrusion Prevention System (IPS) with a preloaded set of attack signatures. The router monitors the network interface (GigabitEthernet0/0) and analyzes all inbound traffic. For each packet, it performs the following steps:

  1. Inspects the packet header and payload for patterns matching known threats.
  2. If a match is found (e.g., ICMP Echo Request matching Sig ID 2004), an alert is generated.
  3. This alert is logged locally in the router‚Äôs console.
  4. The same alert is forwarded to the SYSLOG server for centralized logging and later analysis.

This system does not block the traffic (since it's an IDS, not an IPS), but it provides administrators with timely visibility into possible intrusions or policy violations. It allows for early threat detection and aids in forensic investigation.
